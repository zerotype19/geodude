How Scoring Works

Your overall score (0-100%) is calculated from four weighted components. Each component is first converted to a percentage (points/max × 100), then weighted:

Crawlability: 40% weight

Max 42 points → e.g., 40/42 = 95.2%

  • robots.txt validation and AI bot permissions
  • sitemap.xml presence and accessibility
  • AI crawler access verification
  • Real crawler bonus for live bot visits

Structured Data: 30% weight

Max 30 points → e.g., 11/30 = 36.7%

  • JSON-LD coverage and validation
  • FAQ schema implementation
  • Schema variety and richness
  • Google Rich Results compatibility

Answerability: 20% weight

Max 20 points → e.g., 19.65/20 = 98.3%

  • Title tag optimization
  • H1 tag presence and quality
  • Content depth and structure
  • Question-answer format readiness

Trust: 10% weight

Max 10 points → e.g., 9.49/10 = 94.9%

  • HTTP 2xx status codes
  • Page load performance
  • Security headers and SSL
  • Content accessibility

Formula:

Overall = (Crawl% × 0.4) + (Struct% × 0.3) + (Answer% × 0.2) + (Trust% × 0.1)

Note: In the UI, component scores show both raw points (40/42) AND percentages (95%), with color coding: 🟢 Green (70%+), 🟡 Yellow (40-69%), 🔴 Red (<40%).

Structured Data (30% Weight)

Validates JSON-LD structured data that helps AI engines understand your content.

What We Check:

  • JSON-LD Validation: Syntax and schema.org compliance
  • FAQ Schema: FAQPage markup for Q&A content
  • Organization Schema: Business entity information
  • Schema Variety: Multiple schema types for rich content

Detection Method:

We use real browser rendering (not just HTML parsing) to detect structured data the same way Google and AI engines do:

  • Renders pages with headless Chromium
  • Executes JavaScript to catch dynamic schemas
  • Validates JSON-LD syntax and schema.org compliance
  • Verifies Google Rich Results compatibility

Common Issues:

  • Critical: No structured data found
  • Warning: Invalid JSON-LD syntax
  • Warning: Missing FAQ schema on Q&A pages
  • Warning: Incomplete Organization schema

Answerability (20% Weight)

Measures how well your content can answer user questions and appear in AI-generated responses.

What We Check:

  • Title Tags: Presence, length, and question format
  • H1 Tags: Single H1 per page with descriptive text
  • Content Depth: Sufficient content to answer questions
  • Question Format: Content structured for Q&A

Optimization Tips:

Title Tag Best Practices

  • Include target keywords naturally
  • Use question format when appropriate ("How to...", "What is...")
  • Keep under 60 characters for optimal display
  • Make each title unique across your site

H1 Tag Guidelines

  • One H1 per page (required)
  • Descriptive and keyword-rich
  • Different from the title tag
  • Clear and user-friendly

Common Issues:

  • Critical: Missing title tags
  • Critical: No H1 tags found
  • Warning: Thin content (<300 words)
  • Warning: Duplicate title tags

Trust (10% Weight)

Evaluates technical health and user experience factors that affect AI engine confidence.

What We Check:

  • HTTP Status: 2xx response codes for all pages
  • Load Performance: Page speed and Core Web Vitals
  • Security Headers: HTTPS, security headers
  • Accessibility: Basic accessibility compliance

Issue Severity Levels

Issues are categorized by severity to help you prioritize fixes:

Critical

Major issues that prevent AI crawlers from accessing content or significantly impact visibility:

  • robots.txt blocks AI bots
  • Missing sitemap.xml
  • HTTP 4xx/5xx errors
  • No structured data found
  • Missing title or H1 tags

Warning

Important but non-blocking issues that affect optimization and user experience:

  • Invalid JSON-LD syntax
  • Missing FAQ schema on Q&A pages
  • Thin content (<300 words)
  • Slow page load times
  • Duplicate title tags

Info

Optimization opportunities and minor improvements:

  • Missing meta descriptions
  • Opportunities for additional schema types
  • Content length recommendations
  • Performance optimization suggestions
